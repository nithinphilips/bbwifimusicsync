QScript Documentation

Conventions

[]: Items in brackets are optional.
"": Double quotes are required where shown.

Description

This is a single level scan command:

Scan "http://www.example.com/" [with filter ".jpg"] 
[extract links and resources] [<CHAIN>] save to "C:\Downloads" [rename as "{name}.ext"]
Scan "<url>"
Specifies the URL to scan.
[with] filter "<filter>"
A filter to use on the results. Currently only a substring search on the url is supported. The with part is optional.
extract links and resources
Choose what to extract. Links point to other pages. Resources are things that are shown in the page, like images.
You can enter links or resources or links and resources or resources and links.
save to "<folder>"
Choose the folder to save to final results.
rename as "<pattern>"
The pattern with which to rename files. Currently supported variables are {name}, file name without extension and {ext}, the file extension.
<CHAIN>
You can chain multiple scan commands. A chain should look like this: 
Scan results [with filter ".jpg"] [extract links and resources]
Same rules as the root scan command applies. Only difference is that you have to enter the keyword results instead of a url.

Comments

// This is a single line comment

/*
 * This is a multi line comment
 */

You can also have multiple scan commands in a single script file.

Example

This is a sample script file

/*
 * This Script tests all possible options.
 *
 */
 
 
// Simple script
scan "http://www.example.com/example.html" with filter ".oneFile" extract links save to "C:\Downloads"

// Nested script
scan "http://www.example.com/example.html" with filter ".oneFile" extract links
	scan results with filter "link" extract links
		scan results with filter "jpg" extract links and resources
			scan results with filter "jpg" extract resources save to "C:\Downloads" rename "{name}.{ext}"

// Another nested script with minimal arguments
scan "http://www.example.com/example.html" save to "C:\Downloads"
